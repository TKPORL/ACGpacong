## 2. 数据自动更新机制(讲给以后的你听)

**不是跑在你自己电脑上的,跑在 GitHub 云端。**

| 北京时间 | UTC 时间 | 触发动作 |
|---------|---------|---------|
| 8:00 | 0:00 | GitHub Actions 自动跑 scraper.py |
| 13:00 | 5:00 | 同上 |
| 20:00 | 12:00 | 同上 |

### 完整流程

```
[1] GitHub 闹钟响(8/13/20)
        ↓
[2] GitHub Actions 启动一台 Linux 小电脑(ubuntu-latest)
        ↓
[3] 跑 scraper.py --pages 3 --delay 0.6
        ↓
[4] 生成 data/acgyx_latest.json + dist/data/acgyx_latest.json
        ↓
[5] git add → git commit → git push(自动)
        ↓
[6] Cloudflare Pages 监听到 main 分支更新
        ↓
[7] 自动 build(跑 npx wrangler versions upload)+ 部署
        ↓
[8] acgyx.tkporl.workers.dev 显示新数据
```

### 怎么看跑没跑?

- **GitHub Actions 日志**:https://github.com/TKPORL/ACGpacong/actions
- **Cloudflare Build 日志**:Cloudflare Dashboard → Workers & Pages → acgyx → Deployments

### 可靠性

- **99% 以上准时**(GitHub Actions cron 是行业标准)
- **可能失败的情况**:
  1. GitHub 维护(极罕见,可能延迟 5-30 分钟)
  2. ACG 游戏姬网站改版 → scraper.py 抓不到数据 → 不更新(代码问题,GitHub 跑没跑都没用)

## 3. Cloudflare Pages UI 最终配置(关键!)

进 https://dash.cloudflare.com → Workers & Pages → acgyx → Settings → Builds & deployments

| 字段 | 值 | 说明 |
|------|------|------|
| **Build command** | *(留空)* | 不需要,纯静态 |
| **Deploy command** | `npx wrangler versions upload` | ⚠️ 见下面"为什么是这个" |
| **Non-production branch deploy command** | *(留默认 `npx wrangler versions upload`)* | Cloudflare 强制必须有值,不能空 |
| **Path** | *(留空)* | 默认就是仓库根 |

> **关于 Deploy command 选 `npx wrangler versions upload` 的原因**:
> - 用户最早想用 `npx wrangler pages deploy dist --project-name=acgyx --branch=main`,实测也能跑
> - 但 Cloudflare UI 的 `Non-production branch deploy command` 字段留空就报错"内部错误阻止了表格提交"
> - 实测 `npx wrangler versions upload` 也能成功部署 dist(看 build log: `Read 6 files from the assets directory /opt/buildhome/repo/dist` + `Uploaded acgyx`)
> - 两个命令都能用,既然默认的能跑,就不折腾了

## 4. 这次踩的坑(按时间顺序)

### 坑 1:Build command 填了目录路径

**症状**:`执行用户构建命令:单html版本 /bin/sh: 1: 单html版本: 未找到`

**原因**:误把"Build output directory"的值填到了 Build command 字段。

**修复**:Build command 留空,根 URL 放 `index.html`(即 `dist/index.html`)。

### 坑 2:Cloudflare Pages 新版 UI 没了"Build output directory"

**症状**:UI 上只有 Build command / Deploy command,找不到 Build output directory。

**原因**:Cloudflare Pages 新版(2025+)不再用传统 Pages 部署管线,改成用 wrangler 4.x 部署,默认命令是 `npx wrangler deploy`(Workers 命令)。

### 坑 3:`npx wrangler deploy` 报错 "Cannot use assets with a binding in an assets-only Worker"

**症状**:
```
✘ [ERROR] Cannot use assets with a binding in an assets-only Worker.
Please remove the asset binding from your configuration file, or provide a Worker script in your configuration file (`main`).
```

**原因**:Cloudflare Pages 内部为 wrangler 注入了 assets binding,但 `wrangler deploy`(Workers 模式)需要 `main` 字段指向 Worker 入口脚本,没 main 就报这个错。

**尝试过的修复(都没用)**:
1. 加 `wrangler.jsonc` 配 `main: "src/index.js"` + 空 Worker → 报同样错
2. 加 `wrangler.jsonc` 配 `assets.directory: "./dist"` + 删 binding → 报同样错
3. 加 `wrangler.jsonc` 配 `main` + `assets.directory` + 删 binding → 报同样错

**最终修复**:不管 Deploy command,直接用 Cloudflare 默认的 `npx wrangler versions upload`,**实测能成功部署 dist**。

### 坑 4:中文目录名 wrangler 4.x 不认

**症状**:`assets.directory: "./单html版本"` 报错路径解析失败。

**原因**:wrangler 4.x 对中文/unicode 路径处理有 bug。

**修复**:重命名 `单html版本/` → `dist/`,同步更新:
- `wrangler.jsonc`(已删)
- `.github/workflows/scrape.yml`:`../data/acgyx_latest.json` 输出到 `../dist/data/acgyx_latest.json`
- `.gitignore`:`dist/_shot_*.png` / `dist/_test_shot_*.png`

### 坑 5:GitHub API 速率限制

**症状**:`git log` 没问题,但通过 GitHub REST API 读文件偶尔被限流。

**修复**:改用 `git show origin/main:filename` 直接读 Git 仓库文件,不调 API。

## 6. 故障排查速查表

| 症状 | 可能原因 | 排查方法 |
|------|---------|---------|
| 网站打不开 | Cloudflare 故障 | 直接看 https://acgyx.tkporl.workers.dev/,不是本地浏览器问题 |
| 网站数据旧 | Actions 没跑 / Cloudflare build 失败 | 先看 GitHub Actions 列表 → 失败原因 → 再看 Cloudflare Deployments 列表 |
| Actions 失败 | scraper.py 抓不到 | 看 Actions 日志,可能 ACG 网站改版了,要改 scraper.py |
| Cloudflare build 失败 | Deploy command 错了 | 检查 Settings → Builds & deployments |
| GitHub Actions 没自动跑 | cron 表达式错 | 看 .github/workflows/scrape.yml 第 7-8 行 |
| cron 表达式怎么改? | 北京时间转 UTC 减 8 小时 | 8:00 北京 = 0:00 UTC;13:00 = 5:00 UTC;20:00 = 12:00 UTC |
